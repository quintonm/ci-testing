JDBC Leak Detecter

This patch to p6spy enables you to detect any JDBC resources that have not been properly closed (i.e. Connections, Statements, and ResultSets).

Steps:

  1. Configure P6SPY as your Data Source (see http://www.p6spy.com)
  2. Make sure that commons-lang.jar, p6spy.jar, and p6spy_leak.jar are in your class path.
  3. Configure spy.properties to use the following module: module.log=com.p6spy.engine.leak.P6LeakFactory
  4. Place JDBCLeak.jsp in the root of your web application.
  5. Start your application and exercise it thoroughly for a while.
  6. Run JDBCLeak.jsp to view a stack trace for any JDBC leaks.
  7. Halt the application and remove the leaks.
  8. Repeat steps 5-7 until all leaks are removed.
